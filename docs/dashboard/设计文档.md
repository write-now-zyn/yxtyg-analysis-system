# 整体看板功能设计文档

## 1. 系统架构

### 1.1 技术栈

- 后端：Spring Boot + MyBatis-Plus + MySQL 8
- 前端：Vue.js + Element UI

### 1.2 模块结构

```
后端：
├── controller/DashboardController.java    # 控制器
├── service/DashboardService.java          # 服务接口
├── service/impl/DashboardServiceImpl.java # 服务实现
├── mapper/DashboardMapper.java            # Mapper接口
├── mapper/DashboardMapper.xml             # SQL映射
└── vo/DashboardVO.java                    # 数据传输对象

前端：
├── api/dashboard.js                       # API接口
└── views/Dashboard.vue                    # 页面组件
```

## 2. API设计

### 2.1 获取看板概览数据

**请求**
```
GET /api/dashboard/overview?month=202603
```

**参数**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| month | String | 否 | 月份（yyyyMM格式），默认当前月 |

**响应**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "month": "202603",
    "statistics": {
      "demand": {
        "standard": 6,
        "reachedCount": 8,
        "totalCount": 13,
        "unreachedCities": [
          { "city": "扬州", "count": 5, "reached": false }
        ],
        "ranking": [
          { "city": "南京", "count": 12, "reached": true }
        ]
      },
      "review": { ... },
      "training": { ... }
    }
  }
}
```

## 3. 数据库设计

### 3.1 数据来源表

无需新建表，使用现有表：

| 表名 | 用途 |
|------|------|
| t_work_order | 需求提单统计（order_no前6位=月份） |
| t_review_meeting | 评审参与统计（meeting_time + participants JSON） |
| t_training_report | 培训上报统计（report_month） |
| t_experimenter | 获取所有地市列表 |

### 3.2 SQL查询

**需求提单统计**
```sql
SELECT city, COUNT(*) as count
FROM t_work_order
WHERE LEFT(order_no, 6) = '202603'
GROUP BY city
ORDER BY count DESC
```

**培训上报统计**
```sql
SELECT city, COUNT(*) as count
FROM t_training_report
WHERE report_month = '2026-03'
GROUP BY city
ORDER BY count DESC
```

**评审参与统计**
- 先按meeting_time筛选月份
- 解析participants JSON字段获取参与地市
- 在Java层聚合统计

## 4. 前端设计

### 4.1 组件结构

```
Dashboard.vue
├── 月份选择器 (el-date-picker)
├── 总览卡片行 (el-row > el-col*3)
│   ├── 需求提单卡片
│   ├── 评审参与卡片
│   └── 培训上报卡片
└── 排行榜行 (el-row > el-col*3)
    ├── 需求提单排行
    ├── 评审参与排行
    └── 培训上报排行
```

### 4.2 状态管理

```javascript
data() {
  return {
    loading: false,
    selectedMonth: '202603',  // 当前选中月份
    dashboardData: null       // 后端返回的完整数据
  }
}
```

## 5. 接口调用流程

```
1. 页面加载 → created() 调用 loadData()
2. loadData() → 调用 API getDashboardOverview({ month })
3. API请求 → GET /api/dashboard/overview?month=xxx
4. 后端处理 → DashboardController → DashboardService → DashboardMapper
5. 返回数据 → 更新 dashboardData
6. computed → 计算属性 statistics 响应式更新视图
```
