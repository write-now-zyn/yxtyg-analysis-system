# 模型设置模块设计文档

## 一、功能概述

模型设置模块包含两个子功能：
1. **大模型设置**：配置系统使用的大模型服务，支持多种模型提供商
2. **智能体参数**：为系统中不同的AI能力单独设置请求参数和模型

---

## 二、数据库设计

### 2.1 大模型配置表 (t_model_config)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键ID |
| name | VARCHAR(50) | 配置名称（别名） |
| provider | VARCHAR(50) | 提供商类型（ollama/openai/custom） |
| base_url | VARCHAR(255) | API基础地址 |
| api_key | VARCHAR(255) | API密钥（加密存储） |
| models | TEXT | 可用模型列表（JSON） |
| is_default | TINYINT | 是否默认模型（0-否,1-是） |
| status | TINYINT | 状态（0-禁用,1-启用） |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

### 2.2 智能体配置表 (t_agent_config)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键ID |
| agent_code | VARCHAR(50) | 智能体编码（唯一标识） |
| agent_name | VARCHAR(100) | 智能体名称 |
| description | VARCHAR(500) | 功能描述 |
| model_config_id | BIGINT | 关联的大模型配置ID |
| model_name | VARCHAR(100) | 使用的具体模型名称 |
| temperature | DECIMAL(3,2) | 温度参数（0-2） |
| top_p | DECIMAL(3,2) | Top P参数（0-1） |
| repetition_penalty | DECIMAL(3,2) | 重复惩罚（0-2） |
| system_prompt | TEXT | 系统提示词 |
| max_tokens | INT | 最大输出token数 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

### 2.3 预置智能体数据

| agent_code | agent_name | description |
|------------|------------|-------------|
| participant_parser | 与会人员识别 | 从签到文本中识别与会人员的姓名和地市 |
| training_parser | 转培记录识别 | 从表格文本中提取培训记录信息 |

### 3.2 新增/编辑配置对话框

```
┌─────────────────────────────────────────────────────────────┐
│ 新增大模型配置                                           [X] │
├─────────────────────────────────────────────────────────────┤
│ 配置名称：[________________]                                │
│ 提供商：  [Ollama ▼]                                        │
│ API地址： [________________________] [测试连接]             │
│ API密钥： [________________________] (可选)                 │
│ 默认模型：[________________________] [拉取模型列表]         │
│ 状态：    [○ 启用  ○ 禁用]                                 │
│ 设为默认：[□ 设为默认配置]                                  │
│                                                             │
│                              [取消]  [保存]                 │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 功能说明

1. **新增配置**：
   - 输入配置名称（别名，如"本地Ollama"、"通义千问"）
   - 选择提供商类型（Ollama/OpenAI/自定义）
   - 输入API地址（如 http://localhost:11434）
   - 输入API密钥（如果需要）
   - 点击"测试连接"验证配置是否正确
   - 点击"拉取模型列表"获取可用模型

2. **测试连接**：
   - 发送简单请求验证API是否可用
   - 显示连接状态和响应时间

3. **拉取模型列表**：
   - 调用API获取可用模型列表
   - 存储到models字段（JSON格式）
   - 在智能体配置时可以选择具体模型

### 4.2 编辑智能体参数对话框

```
┌─────────────────────────────────────────────────────────────┐
│ 编辑智能体参数 - 与会人员识别                            [X] │
├─────────────────────────────────────────────────────────────┤
│ 智能体名称：与会人员识别（只读）                             │
│ 功能描述：从签到文本中识别与会人员的姓名和地市（只读）       │
│                                                             │
│ 大模型配置：[本地Ollama ▼]                                  │
│ 使用模型：  [qwen3:0.6b ▼]                                  │
│                                                             │
│ ──────────── 模型参数 ────────────                         │
│ Temperature：[0.7____] (范围: 0-2，控制输出随机性)          │
│ Top P：      [0.9____] (范围: 0-1，核采样参数)              │
│ 重复惩罚：   [1.1____] (范围: 0-2，防止重复输出)            │
│ 最大Token：  [2048___] (最大输出长度)                       │
│                                                             │
│ ──────────── 系统提示词 ────────────                       │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 从文本中提取与会人员的姓名和地市。                       │ │
│ │ 江苏省地市：南京、苏州、无锡、常州、镇江、扬州...       │ │
│ │ ...                                                     │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                             │
│                              [取消]  [保存]                 │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 功能说明

1. **选择大模型配置**：
   - 从已配置的大模型列表中选择
   - 选择后自动加载该配置下的可用模型

2. **选择具体模型**：
   - 从选定的大模型配置中拉取的模型列表中选择
   - 也可以手动输入模型名称

3. **参数调整**：
   - **Temperature**：控制输出随机性，0最确定，2最随机
   - **Top P**：核采样参数，控制输出多样性
   - **重复惩罚**：防止模型重复输出相同内容
   - **最大Token**：限制输出长度

4. **系统提示词**：
   - 可编辑的系统提示词
   - 支持多行文本输入
   - 提供默认值，用户可根据需要调整

## 五、后端接口设计

### 5.1 大模型配置接口

```
GET    /api/model-config/list        # 获取配置列表
POST   /api/model-config/add         # 新增配置
PUT    /api/model-config/update      # 更新配置
DELETE /api/model-config/delete/{id} # 删除配置
POST   /api/model-config/test/{id}   # 测试连接
POST   /api/model-config/fetch-models/{id} # 拉取模型列表
POST   /api/model-config/set-default/{id}  # 设置默认配置
```

### 5.2 智能体配置接口

```
GET    /api/agent-config/list        # 获取智能体列表
PUT    /api/agent-config/update      # 更新智能体参数
GET    /api/agent-config/detail/{code} # 获取智能体详情
```

## 六、实现优先级

1. **第一阶段**：数据库表创建 + 后端接口实现
2. **第二阶段**：大模型设置页面实现
3. **第三阶段**：智能体参数页面实现
4. **第四阶段**：集成到现有AI服务中

---

## 七、注意事项

1. API密钥需要加密存储，不能明文保存
2. 测试连接时需要设置超时时间，避免长时间等待
3. 智能体编码是系统预置的，用户不能新增或删除智能体
4. 系统提示词支持变量替换，如 `{cities}` 自动替换为地市列表
5. 当修改智能体参数后，需要清除相关的缓存
