## 2. 开发环境搭建

### 2.1 后端环境
1. 安装JDK 8
2. 安装Maven 3.6+
3. 导入IDE（IDEA或Eclipse）
4. 执行Maven依赖下载
5. 运行YxtygApplication.java

### 2.2 前端环境
1. 安装Node.js 14+
2. 进入yxtyg-web目录
3. 执行 `npm install`
4. 执行 `npm run serve` 启动开发服务器

### 2.3 数据库初始化
1. 连接到MySQL数据库（172.26.55.83:3306）
2. 执行 `docs/init.sql` 脚本

## 3. 接口调用示例

### 3.1 获取体验官列表
```javascript
// GET /api/experimenter/list?current=1&size=10
import { getExperimenterList } from '@/api/experimenter'

getExperimenterList({ current: 1, size: 10 }).then(res => {
  console.log(res.data.records)
})
```

### 3.2 导入工单
```javascript
// POST /api/workorder/import
import { importWorkOrder } from '@/api/workorder'

const formData = new FormData()
formData.append('file', file)
importWorkOrder(file).then(res => {
  console.log('导入结果：', res.data)
})
```

### 3.3 新增评审会议纪要
```javascript
// POST /api/review/add
import { addReview } from '@/api/review'

const data = {
  demandNo: 'REQ-2026-001',
  title: '体验官需求评审会',
  meetingTime: '2026-03-18 14:00:00',
  participants: JSON.stringify([
    { name: '张三', city: '南京' },
    { name: '李四', city: '苏州' }
  ])
}
addReview(data).then(res => {
  console.log('添加成功')
})
```

## 5. 开发规范

### 5.1 代码规范
- 使用中文注释（专有名词除外）
- 类名使用大驼峰命名
- 方法名使用小驼峰命名
- 常量使用全大写下划线命名

### 5.2 提交规范
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式调整
- refactor: 重构
- test: 测试相关

## 6. 部署说明

### 6.1 后端部署
```bash
# 打包
cd yxtyg-admin
mvn clean package -DskipTests

# 运行
java -jar target/yxtyg-admin-1.0.0.jar
```

### 6.2 前端部署
```bash
# 打包
cd yxtyg-web
npm run build

# 部署dist目录到nginx
```

### 6.3 Nginx配置示例
```nginx
server {
    listen 80;
    server_name localhost;

    location / {
        root /path/to/yxtyg-web/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 7. 常见问题

### Q1: 启动报数据库连接失败
检查数据库连接配置是否正确，确保数据库服务已启动。

### Q2: 前端跨域问题
开发环境已配置代理，生产环境需配置Nginx反向代理。

### Q3: 文件上传大小限制
默认限制50MB，可在application.yml中调整：
```yaml
spring:
  servlet:
    multipart:
      max-file-size: 50MB
      max-request-size: 50MB
```
