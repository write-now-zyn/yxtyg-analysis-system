# 一线体验官专项数据分析系统 - 需求文档

## 1. 项目背景

### 1.1 项目概述
本项目为江苏移动省公司一线体验官专项的数据分析平台，旨在实现需求统计、评审统计、转培上报统计及体验官管理四大核心功能，提升数据管理效率和分析能力。

### 1.2 目标用户
- 江苏移动省公司管理人员
- 各地市体验官接口人
- 系统管理员

## 2. 功能需求

### 2.1 需求统计模块

#### 2.1.1 功能描述
定期上传工单表格，系统自动进行数据处理和存储。

#### 2.1.2 功能点
1. **Excel导入**
   - 支持上传Excel格式的工单表格
   - 按工单号与已存储内容进行去重
   - 自动格式化数据并保存

2. **数据关联**
   - 需求提交人必须为体验官列表中的成员
   - 自动关联体验官信息

3. **多条件搜索**
   - 支持按提出人姓名搜索
   - 支持按年月搜索
   - 支持按地市搜索
   - 支持关键词搜索（标题、内容、关键词字段）

4. **详情查看**
   - 弹窗展示工单完整信息

5. **Excel导出**
   - 支持将搜索结果导出为Excel文件

### 2.2 评审统计模块

#### 2.2.1 功能描述
录入和管理评审会议纪要，支持多维度的查询和导出。

#### 2.2.2 功能点
1. **会议纪要录入**
   - 必填项：评审方案需求号、标题、会议时间、与会人列表
   - 与会人包含：姓名、地市
   - 支持动态添加多个与会人

2. **列表展示**
   - 展示所有会议纪要
   - 显示：需求号、标题、会议时间、与会人列表

3. **多条件搜索**
   - 支持按会议时间范围搜索
   - 支持按与会人员姓名搜索

4. **数据维护**
   - 支持新增、编辑、删除操作

5. **Excel导出**
   - 支持将搜索结果导出为Excel文件

### 2.3 转培上报统计模块

#### 2.3.1 功能描述
按地市维度管理培训上报记录，包含培训明细和培训截图。

#### 2.3.2 功能点
1. **培训上报录入**
   - 按地市+月份维度上报
   - 每次上报包含8条以上培训记录

2. **培训记录信息**
   - 培训时间
   - 组织人员
   - 培训主体
   - 覆盖人数
   - 培训内容
   - 培训截图（保存到数据库）

3. **列表展示**
   - 按地市+月份汇总展示
   - 显示：地市、上报月份、培训记录数

4. **多条件搜索**
   - 支持按地市搜索
   - 支持按月份搜索
   - 支持按培训内容关键词搜索
   - 支持按培训主体关键词搜索

5. **详情查看**
   - 弹窗展示该地市当月所有培训记录
   - 支持查看培训截图

6. **Excel导出**
   - 支持将搜索结果导出为Excel文件

### 2.4 体验官管理模块

#### 2.4.1 功能描述
管理各地市体验官信息，支持接口人标记和筛选。

#### 2.4.2 功能点
1. **体验官信息管理**
   - 地市（必填）
   - 姓名（必填）
   - 电话（必填）
   - 邮箱（选填）
   - 角色（选填）
   - 备注

2. **接口人管理**
   - 支持标记各地市接口人
   - 支持筛选接口人

3. **多维度搜索**
   - 支持按地市搜索
   - 支持按姓名搜索
   - 支持按电话搜索
   - 支持按是否接口人筛选

4. **数据维护**
   - 支持新增、编辑、删除操作

5. **Excel导出**
   - 支持将搜索结果导出为Excel文件

### 2.5 公共功能

#### 2.5.1 数据导出
- 所有模块均支持Excel导出
- 导出内容与当前搜索条件一致

#### 2.5.2 数据关联
- 需求提交人必须关联体验官表中的成员
- 导入时自动匹配体验官信息

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间 < 3秒
- 数据查询响应时间 < 2秒
- Excel导入支持批量处理

### 3.2 安全需求
- 数据库访问权限控制
- 接口访问日志记录

### 3.3 可用性需求
- 界面简洁直观，易于操作
- 操作有明确的反馈提示

## 4. 约束条件

### 4.1 技术约束
- 前端：Vue2 + ElementUI
- 后端：Java8 + SpringBoot 2.x
- 数据库：MySQL 8.0
- 缓存：Redis

### 4.2 数据约束
- Excel文件最大50MB
- 培训截图单张最大5MB
- 支持的图片格式：jpg、png

## 5. 交付物

1. 可运行的前后端代码
2. 数据库初始化脚本
3. 项目文档（需求、设计、开发、测试）
